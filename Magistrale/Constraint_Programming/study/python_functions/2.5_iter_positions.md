# 2.5 — `iter_positions` (strategie di rimozione)

**Cosa fa:** decide l'**ordine** in cui le 81 celle vengono provate per la rimozione. Tre strategie da confrontare (richiesto dalla specifica).

## Il codice

```python
def iter_positions(strategy, rng):
    positions = [(r, c) for r in range(9) for c in range(9)]

    if strategy == "random":
        rng.shuffle(positions)
        return positions

    if strategy == "symmetry":
        seen = set()
        ordered = []
        for r, c in positions:
            mirror = (8 - r, 8 - c)
            key = tuple(sorted(((r, c), mirror)))
            if key in seen:
                continue
            seen.add(key)
            ordered.append((r, c))
        rng.shuffle(ordered)
        return ordered

    if strategy == "density":
        return sorted(positions,
                      key=lambda item: abs(item[0]-4) + abs(item[1]-4),
                      reverse=True)
```

## Le tre strategie

### `random`
Permutazione casuale di tutte le 81 celle. Baseline.

### `symmetry` — simmetria centrale
Rimuove le celle a **coppie centrali**: per `(r,c)`, il mirror è `(8-r, 8-c)`.

```
key = tuple(sorted(((r,c), mirror)))  →  chiave canonica della coppia
                                          (r,c) e il suo mirror hanno la STESSA chiave
                                          → se ne tiene un solo rappresentante
```

```
caso speciale: centro (4,4)
   mirror = (8-4, 8-4) = (4,4) = se stesso  →  autosimmetrico
   key = ((4,4),)  →  incluso una sola volta, nessuna seconda rimozione
```

Nel loop di generazione, rimuovere `(r,c)` rimuove **anche** il mirror → 2 celle per decisione.

### `density`
Ordina per **distanza di Manhattan dal centro (4,4)**, decrescente.

```
angoli (0,0),(0,8),(8,0),(8,8)  →  distanza 8  →  rimossi per primi
centro (4,4)                    →  distanza 0  →  rimosso per ultimo
```

Ipotesi di partenza: le celle periferiche hanno meno vincoli incidenti, quindi più facili da rimuovere.

## Risultati empirici (da sapere)

| Strategia | Clue medi | Nota |
|---|---|---|
| `random` | **~24.6** | i migliori (meno clue) |
| `density` | ~25.4 | l'ipotesi "angoli meno vincolati" NON regge |
| `symmetry` | il più alto | paga una "double tax": ogni rifiuto blocca 2 celle |

## Domande probabili

- *"Perché simmetria centrale e non altre?"* → È quella dei Sudoku editoriali (The Times). Le riflessioni orizzontali/verticali si usano meno; le rotazioni di 90° non si applicano perché l'ordine delle cifre nei blocchi non è invariante per rotazione.
- *"Density risparmia davvero?"* → No (25.4 vs 24.6). La struttura del Sudoku rende tutte le celle ugualmente vincolate. Il valore è il **confronto**, non la performance.
- *"Perché symmetry ha più clue?"* → Rimuove a coppie: ogni rifiuto blocca 2 celle invece di 1, quindi rimuovi statisticamente meno.

## Frase secca per l'orale

> *"Tre strategie di ordinamento delle celle da rimuovere: random (baseline), symmetry (coppie centro-simmetriche, alla The Times), density (periferia prima, per distanza di Manhattan dal centro). Empiricamente random vince (~24.6 clue), density non aiuta, symmetry produce più clue per la double-tax delle coppie. Il valore richiesto dalla specifica è il confronto tra strategie, non vincere."*
